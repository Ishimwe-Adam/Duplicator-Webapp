# 🚀 DUPLICATOR LTD — FULL-STACK WEB APPLICATION BUILD PROMPT

---

## 🎯 PROJECT OVERVIEW

You are a world-class senior full-stack engineer and UI/UX designer. Your mission is to transform the existing **Duplicator Ltd** marketing website (https://ecom-site-builder--ishimweadam0.replit.app/) into a **complete, production-grade, fully responsive web application** — a powerful internal company platform combined with a client-facing portal. This must be so impressive that it stands out globally.

**Company:** Duplicator Ltd  
**Industry:** Premium Printing · Branding · Sewing Solutions  
**Location:** Kigali, Rwanda  
**Tagline:** *Switch Your Brand ON*  
**Currency:** All prices must be displayed in **FRW (Rwandan Francs)** — research realistic market pricing for each product/service category and apply it. Remove any WhatsApp icons or chat badges from product cards entirely.

---

## 🏗️ TECH STACK (Use Best Practices)

**Frontend:**
- React.js (with TypeScript) or Next.js 14+ (App Router preferred)
- Tailwind CSS + shadcn/ui components
- Framer Motion for animations
- React Query / TanStack Query for data fetching
- Recharts or Chart.js for analytics dashboards

**Backend:**
- Node.js + Express.js OR Next.js API routes
- PostgreSQL (primary database via Prisma ORM)
- Redis (for caching and session management)
- JWT + Refresh Token authentication
- Bcrypt for password hashing
- Nodemailer (email notifications)
- Multer + Cloudinary (file/image uploads)

**Infrastructure:**
- RESTful API architecture (fully documented)
- Role-Based Access Control (RBAC)
- Input validation (Zod or Joi)
- Rate limiting and security middleware (Helmet, CORS, express-rate-limit)
- Environment variables via `.env`
- Responsive on all devices: mobile, tablet, desktop

---

## 🎨 DESIGN DIRECTION

**Aesthetic:** Luxury-Industrial. Think bold typography, deep navy/charcoal backgrounds with vibrant accent colors (electric blue `#0066FF` and warm gold `#FFB800`). Clean geometric layouts with sharp shadows. Premium feel — like a global brand agency.

- **Fonts:** Use `Syne` (display/headings) + `DM Sans` (body) from Google Fonts
- **Color Palette:**
  - Primary: `#0A0F2C` (deep navy)
  - Accent: `#0066FF` (electric blue)
  - Gold: `#FFB800`
  - Surface: `#F5F7FF`
  - Text: `#1A1A2E`
- Smooth page transitions, hover micro-interactions, skeleton loaders
- Sidebar navigation with collapsible menu for dashboards
- Glassmorphism cards for stats/KPIs
- Dark mode toggle (localStorage persisted)

## 🔐 MODULE 1 — AUTHENTICATION SYSTEM

Build a complete, secure authentication system:

- **Registration:** Name, email, phone, password, role assignment (admins assign roles)
- **Login:** Email + password with JWT access token (15min) + refresh token (7 days, HttpOnly cookie)
- **Multi-factor Authentication (MFA):** Optional TOTP (Google Authenticator compatible)
- **Password Reset:** Email OTP flow with 10-minute expiry
- **Session Management:** Active sessions list with ability to revoke
- **Account Lock:** Lock after 5 failed login attempts (30-min cooldown)
- **Audit Log:** Every login attempt logged with IP, timestamp, device
- Beautiful, branded login/signup pages matching Duplicator Ltd identity
- "Remember Me" functionality
- Profile picture upload

## 📋 DATABASE MODELS (Prisma Schema)

Design and implement all models including:
- `User` (id, name, email, password, role, phone, avatar, isActive, createdAt)
- `Client` (extends User with company details)
- `Order` (id, clientId, serviceType, status, specs, deadline, assignedTo, totalPrice, files)
- `Invoice` (id, invoiceNumber, clientId, lineItems, vatAmount, total, status, paidAt)
- `Task` (id, title, description, assigneeId, priority, status, dueDate, createdBy)
- `Message` (id, senderId, receiverId, channelId, content, attachments, readAt)
- `Channel` (id, name, type, members)
- `Notification` (id, userId, type, message, readAt, link)
- `Quote` (id, clientId, description, files, estimatedPrice, status, respondedAt)
- `Document` (id, name, url, folderId, uploadedBy, sharedWith)
- `AuditLog` (id, userId, action, entity, entityId, ipAddress, createdAt)
- `Product` (id, name, category, description, priceFRW, minQty, deliveryDays, images)
- `Announcement` (id, title, body, priority, authorId, createdAt)
